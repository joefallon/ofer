# CONTRIBUTING

Thank you for your interest in contributing to OFER.

OFER is intended to be a lawful, durable, community-driven open fantasy roleplaying project. To keep the project healthy, contributors must be careful about source provenance, licensing, originality, clarity, and playability.

## Core Rule

Only submit material that you have the right to contribute.

That means your submission should be one of the following:

- your own original work
- a lawful adaptation of a source that is clearly compatible with this repository's licensing and content rules
- an edit or improvement to existing repository content made in compliance with the repository license

## What You Must Not Submit

Do not submit:

- text copied from closed or proprietary rulebooks
- text copied from websites, PDFs, or VTT modules without a clearly compatible open license
- copied setting lore, monster writeups, subclasses, spells, or rules text from non-open products
- protected logos, trade dress, or branding
- artwork you do not have the right to contribute
- material with unclear or uncertain provenance

If you are not sure whether a source is safe to use, do not submit it until the licensing is confirmed.

## Current Source Policy

At present, OFER is using material derived from:

- Black Flag Reference Document 1.0 (BFRD 1.0)

Only sources actually used by the project are attributed in `ATTRIBUTIONS.md`.

If the project later adopts additional compatible sources, those sources will be added to the attribution and notice files.

## Contribution License

By submitting a contribution to this repository, you represent that:

1. you have the legal right to submit the material
2. the material is original to you, or lawfully derived from compatible open material
3. the material does not knowingly infringe the rights of others
4. you agree that your contribution may be distributed as part of this project under the repository's license terms

Unless explicitly stated otherwise by the maintainers for a specific contribution area, contributions accepted into this repository are intended to be distributed under CC BY 4.0.

## Attribution and Provenance

This repository uses centralized attribution at the repository level.

Contributors are not expected to provide paragraph-by-paragraph source tracing. However, if your contribution is derived from a compatible open source, you should clearly say so in your pull request description so maintainers can determine whether the repository attribution files need to be updated.

A good pull request note might say:

- "This section is original."
- "This section is adapted from BFRD 1.0 and rewritten substantially for clarity."
- "This monster is an original design intended to fill a gameplay niche."

## AI-Assisted Contributions

AI may be used for brainstorming, outlining, rewriting, formatting, and drafting.

AI use does not excuse poor sourcing or legal problems.

AI-assisted contributions are judged by the same standards as any other contribution:

- lawful to include
- sufficiently original for the project
- clear
- useful
- well written
- good for play

Do not use AI to launder closed or proprietary text into the project.

## Design Standards

When contributing, prefer:

- clarity over cleverness
- usability over ornament
- original expression over imitation
- strong gameplay purpose over filler
- consistent terminology and formatting
- design that improves actual play

OFER is meant to be a living game, but not a dumping ground. A contribution should earn its place.

## Pull Request Guidance

When opening a pull request, please include:

- a short summary of the change
- whether the submission is original or adapted
- any source and license information relevant to the submission
- any design rationale that helps reviewers understand the change
- any playtest notes, if applicable

## Maintainer Rights

Maintainers may:

- request revisions
- reject contributions
- edit accepted contributions for style, clarity, structure, or compliance
- remove content later if legal, attribution, quality, or scope concerns arise

## Forks

Forks are welcome.

OFER is designed to be a community-friendly open project. If your design goals differ from the main branch, you are encouraged to explore them in a fork rather than forcing unrelated changes into core.

## Questions

If you are unsure whether something is safe or appropriate to submit, open an issue first and ask before spending a lot of time on the pull request.

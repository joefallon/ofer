## Multiclass Class Features

Multiclassing lets you combine class abilities, but it does not let every similar feature stack freely.

When you gain a new level in a class, you gain the features that class grants at that level. In most cases, that is the whole rule. If you take a level in wizard, you gain the wizard features for that level. If you take a level in fighter, you gain the fighter features for that level.

However, a few features need extra rules when more than one class can grant them. Without those rules, some combinations would become confusing or unintentionally much stronger than intended.

This section covers the general rule first, then the three special cases called out here:

- Channel Divinity
- Multiattack
- Unarmored Defense

## The General Rule

When you gain a level in a class, you gain the class features for that level.

You do not gain the class's starting equipment.

You also do not go back and gain features from earlier levels that you never took. You only gain what belongs to the new class level you actually received.

Example: A fighter 4 who takes a level in rogue becomes fighter 4/rogue 1. That character gains the 1st-level rogue features, not the 5th-level fighter features.

In most multiclass builds, this rule handles everything cleanly. You simply track each class separately and take the features each class level provides.

## When Similar Features Overlap

Some classes grant abilities that resemble each other closely enough that they need special handling.

The guiding idea is this: multiclassing broadens your options, but it usually does not let duplicate class features stack into a larger version unless the rules explicitly say so.

That principle matters most for Channel Divinity, Multiattack, and Unarmored Defense.

## Channel Divinity

If you already have the Channel Divinity feature and gain it again from a different class, you gain the new Channel Divinity options from that class, but you do not gain an additional use unless one of your class levels explicitly increases your uses.

In other words:

- your list of Channel Divinity effects can expand
- your number of uses does not combine across classes

You track uses according to the class feature progression that grants them. If one class gives you more uses at a higher class level, you benefit from that increase. You do not add together separate pools from different classes.

### Example: Cleric and Paladin

A character is cleric 6/paladin 4.

Because the character has reached a cleric level high enough to have two uses of Channel Divinity between rests, the character can use Channel Divinity twice.

The paladin levels may add additional Channel Divinity options, but they do not add more uses on top of the cleric uses.

Whenever the character uses Channel Divinity, they can choose any Channel Divinity effect available from either class.

### What This Means in Play

Multiclassing into another Channel Divinity class makes the feature broader, not automatically deeper.

You get more choices, but not an extra pool of uses just because two classes grant the feature.

## Multiattack

If you gain the Multiattack feature from more than one class, those features do not add together to give you more attacks.

You use the version of Multiattack that your classes allow, but you do not combine them into a larger total.

### Example: Fighter and Monk

A character is fighter 9/monk 5.

Suppose the fighter levels grant Multiattack allowing three attacks, while the monk levels grant Multiattack allowing two attacks.

That character does not make five attacks. The features do not stack that way.

Instead, the character uses the best applicable version available from their class features.

### Why This Rule Exists

Without this rule, multiclass characters could stack similar attack-progress features from several classes into extremely large numbers of attacks. The game does not treat Multiattack that way.

Multiclassing can expand your toolkit, but duplicate attack features do not pile together.

## Unarmored Defense

If you already have Unarmored Defense and gain it again from another class, you do not gain it a second time.

You use only one version of Unarmored Defense, not both.

If different classes would grant different formulas for calculating your Armor Class while unarmored, you do not combine those formulas. You use the one available from the class that gave you the feature first.

### Example: Barbarian and Monk

A character has levels in both barbarian and monk.

Both classes can grant Unarmored Defense, but the character does not gain two separate versions to stack or swap freely under this rule.

The character uses the Unarmored Defense feature from whichever class they took at 1st level.

### Why This Rule Matters

Unarmored Defense is a complete alternative way to calculate Armor Class. It is not a bonus that gets added on top of another AC formula.

If a character could stack multiple versions, the result would quickly become unreasonable. Multiclassing does not allow that.

## The Broader Pattern

When multiclassing, class features usually fall into one of two categories:

### Features That Simply Add On

These work normally. If a new class grants a new feature, you gain it and can use it as written.

### Features That Overlap

If two classes grant versions of the same kind of feature, special multiclass rules may prevent them from stacking.

That distinction is worth remembering whenever you build a multiclass character. Many features combine well. Some only broaden your choices rather than increasing raw power.

## Example: A Multiclass Feature Package

Darian is a cleric 5/fighter 5.

When Darian takes another cleric level and becomes cleric 6/fighter 5, Darian gains the class features of cleric 6.

If one of those features increases Channel Divinity uses, Darian benefits from that increase. If Darian later gains another class that also grants Channel Divinity, Darian may gain additional Channel Divinity options, but not a second separate pool of uses unless a class level explicitly grants more.

If Darian later gained a class feature resembling Multiattack from another class, that would not stack with fighter Multiattack to produce a combined super-version.

Each feature follows its own multiclass rule.

## Common Mistakes to Avoid

A few mistakes show up often with multiclass features:

- assuming that gaining the same named feature twice automatically increases uses
- adding together separate Multiattack progressions from different classes
- trying to combine two versions of Unarmored Defense
- forgetting that multiclassing grants the new class level's features, but not starting equipment

When in doubt, check whether the feature is meant to stack or whether the multiclass rules give it special handling.

## Quick Reference

When you gain a level in a new class:

- you gain that class level's features
- you do not gain starting equipment

Special multiclass feature rules:

- Channel Divinity: gain new options, but not extra uses unless a class level explicitly grants them
- Multiattack: versions from different classes do not add together
- Unarmored Defense: if you gain it from more than one class, you use only the version from the class you took first

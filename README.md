# Open Fantasy Engine for Roleplaying (OFER)

OFER is an open fantasy roleplaying engine built for long-term access, lawful reuse, community iteration, and table-ready play.

This project exists to provide a durable, openly maintained fantasy RPG rules corpus that can be read, improved, versioned, forked, and used at the table without dependence on a closed platform or revocable web-only access model.

OFER is being built from open-licensed foundations and original contributions. The goal is not to imitate any closed product line verbatim. The goal is to create a lawful, durable, open fantasy game that feels familiar to players of modern d20 fantasy roleplaying while remaining fully usable as its own project.

## Project Goals

OFER aims to be:

- open
- lawful
- durable
- community-driven
- VTT-friendly
- readable
- patchable
- forkable
- useful in actual play

More specifically, OFER is intended to support:

- long-term access to the rules without dependence on a single vendor platform
- a public, version-controlled rules corpus
- collaborative improvement through issues, discussion, and pull requests
- use at the table and in VTT environments
- a recognizable fantasy d20 play experience built from open material and original design

## Design Philosophy

The project is guided by a few core principles:

### 1. Open where possible
OFER is built from open-licensed materials and original contributions. The project avoids closed, proprietary, or uncertain-source content.

### 2. Durable over convenient
The rules should be available in normal files that can be stored, backed up, read locally, and maintained over time.

### 3. Community improvement over static release
Rules text is allowed to evolve. If wording is unclear, mechanics are weak, or better implementations emerge through play, the project can improve.

### 4. Clear at the table
Rules should be written in a form that is easy to read, reference, and use during play.

### 5. Original expression matters
OFER may occupy familiar gameplay niches, but it should do so with its own wording, structure, presentation, and design identity.

## Scope

OFER is a fantasy roleplaying engine and rules project.

This repository may eventually include:

- core rules
- classes
- subclasses
- spells
- monsters
- equipment
- conditions
- optional rules
- GM-facing procedures
- VTT-friendly reference text
- original supplementary content

Not every future addition must be present at the start. OFER is expected to grow iteratively.

## Content Rules

The following material is welcome:

- original writing
- original mechanics
- original monsters
- original subclasses
- original setting-neutral support text
- lawful adaptations of compatible open-licensed source material
- editorial rewrites for clarity, consistency, and usability
- AI-assisted content that meets project standards

The following material must not be added:

- text copied from closed or proprietary rulebooks
- text copied from websites or products without a clearly compatible license
- protected logos, trade dress, or branding belonging to other publishers
- copied art without a clearly compatible license
- near-verbatim reproductions of non-open material
- submissions with unclear provenance

If questionable material is discovered later, maintainers may revise or remove it.

## AI Policy

AI may be used in this project as a drafting, editing, brainstorming, formatting, or design-assistance tool.

AI-generated material is not automatically accepted or rejected. It is judged by the same standard as any other submission:

- is it lawful to include
- is it original enough for this project
- is it clear
- is it useful
- is it well written
- is it good game design

Low-quality, derivative, unclear, or legally questionable material should be rejected regardless of whether it was written by a person or generated with AI assistance.

## Compatibility Philosophy

OFER is intended to support familiar fantasy d20 play.

That does not mean this project republishes closed content, claims official status, or attempts to reproduce proprietary books line-for-line. OFER should be understood as its own open fantasy roleplaying project built from open foundations and original work.

## Community Model

OFER is intended to function as a living project.

That means:

- issues can identify unclear wording, design flaws, or missing content
- pull requests can propose fixes and improvements
- contributors can discuss revisions openly
- maintainers can merge improvements into the core project
- forks are welcome for groups that want to take the engine in different directions

The main branch should remain coherent, readable, and playable. Community contribution is encouraged, but quality control still matters.

## Editorial Standards

When editing or extending OFER, prefer:

- clarity over cleverness
- table usability over ornate prose
- explicit wording over ambiguity
- strong structure over repetition
- original expression over imitation
- useful mechanics over novelty for its own sake

A rule should earn its place by making the game better.

## Contributing

Please read `CONTRIBUTING.md` before opening a pull request.

In general, contributors should be prepared to confirm that:

- they have the right to submit the material
- the material is original or derived only from compatible open sources
- the submission does not include closed or uncertain-source content
- the submission is intended to be licensed under this project's license terms

## Why This Project Exists

Many players want a fantasy RPG they can:

- read locally
- back up
- reference during play
- improve over time
- share lawfully within the bounds of its license
- rely on without fear that access will disappear behind a platform decision

OFER exists to help meet that need.
